# Technical Architecture & Pipeline Plan (PLAN)
## Multi-Agent Execution Framework

The project utilizes a 3-Agent architecture designed to synthesize qualitative research into quantitative safety metrics.

---

### 1. Architectural Overview
The pipeline is designed as an automated research synthesizer that ingests unstructured academic data, extracts precise neurocognitive metrics, and applies them to a mathematical pedestrian-crossing scenario. 

This model demonstrates how laboratory-measured cognitive timing deficits translate directly into real-world traffic risks for older pedestrians.

---

### 2. Detailed Agent Roles and Behaviors

#### Agent 1: Research Ingestion & Standardization Agent
- **Objective:** Ingest and parse the 20 primary literature sources (published between 1998 and 2026) to establish a standardized database.
- **Input:** Tracking data from literature matrix and academic text blocks.
- **Processing & Logic:**
  - Standardizes study metadata (Authors, Year, Title, Category).
  - Classifies studies into three designated categories:
    - *Category 1:* Isolated older adult timing capabilities.
    - *Category 2:* Developing and younger populations.
    - *Category 3:* Direct comparative lifespans.
  - Registers global dataset totaling 21,114 participants.
- **Output:** Structured research database profiles.

#### Agent 2: Cognitive Metrics Extractor Agent
- **Objective:** Isolate specific cognitive parameters, experimental paradigms, and behavioral outcomes across the lifespan.
- **Input:** Standardized research database generated by Agent 1.
- **Processing & Logic:**
  - Identifies timing paradigms (Prospective vs. Retrospective).
  - Extracts target interval durations (milliseconds, seconds, or minutes).
  - Identifies core cognitive mediators (processing speed, working memory capacity, attentional control).
  - Pulls explicit timing directional biases (e.g., systematic temporal underestimation).
- **Output:** Parameterized database mapping cognitive deficits and empirical temporal biases.

#### Agent 3: Ecological Safety Mapper & Modeling Agent
- **Objective:** Translate abstract laboratory timing errors into real-world pedestrian crossing risks.
- **Input:** Extracted cognitive parameters and gait speed parameters.
- **Processing & Logic:**
  - Implements the perceived duration formula:
    $$T_{\text{perceived}} = T_{\text{ttc}} \times (1 - \alpha - \gamma)$$
  - Models scenarios across variable attentional loads and environmental conditions (optimal daylight, visual degradation, dual-task failure).
  - Maps structural failure thresholds where internal temporal misjudgment causes premature crosswalk entry.
- **Output:** Analytical scenario comparison matrices and TikZ architectural visualizations.

---

### 3. Execution Pipeline & Data Flow

1. **Phase I (Ingestion):** Ingestion Agent processes research inputs -> Validates database integrity.
2. **Phase II (Extraction):** Extractor Agent maps paradigms -> Generates multidimensional array of cognitive metrics.
3. **Phase III (Modeling):** Safety Mapper Agent computes mathematical crossing decision boundaries -> Synthesizes parameter comparison matrices.
4. **Phase IV (Document Generation):** Injects generated TikZ diagrams and syntheses into the LaTeX template (`main.tex`) to produce the final 33+ page bilingual review paper.

---

### 4. Technical Stack
- **LaTeX Compiler:** LuaLaTeX / XeLaTeX.
- **Document Class:** `custom_report.cls` (12pt, A4 paper, margins: Top/Bottom 2.5cm, Left/Right 3.0cm).
- **Bilingual Engine:** `babel` package with `bidi=basic` mapping Arial system fonts.
- **Graphics Engine:** TikZ (`positioning`, `shapes.geometric` libraries).
